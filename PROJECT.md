# Project Description

## Objective
Implement Azure DevOps capabilities for a new or existing project, including CI/CD pipelines, secure configuration, and optional project management features.

## Requirements

### Mandatory Components
- **Code Integration (5 pts)**  
  Use a version control system (typically Git).

- **Build Pipeline (10 pts)**  
  Implement an automated build process using Azure Pipelines.

- **Release Pipeline (5 pts)**  
  Implement a deployment/release pipeline.

- **Security / Service Connections (10 pts)**  
  - Use secure methods for handling secrets (e.g., Azure Key Vault, service connections).  
  - **No passwords, secrets, or keys may be stored in Git or directly in pipeline definitions.**

### Optional Components
- **Azure Boards** for task tracking and TODO management  
- **Azure Wiki** for project documentation  

## Functional Requirement
- The application must be functional. Non-working code will result in point deductions.

## Documentation

- **Team Members**: Roles and responsibilities of each member

- Recommended: **4–5 members**
- Any deviation (smaller or larger teams) requires prior approval
